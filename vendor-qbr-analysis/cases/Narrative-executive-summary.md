## ナラティブ（Narrative）とは、

- 「データや事実をストーリーとしてつなぎ、読み手を理解と合意形成に導くための説明方法」 のこと

### もう少し具体的に

- 同じ内容でも、単に事実を並べた説明 と意味のあるストーリーとしてまとめた説明（ナラティブ）では、相手の理解度と共感に大きな差が出ます。

#### 事実だけの説明（ナラティブなし）

- A社は平均4.1点、B社は3.9点でした。
- ANOVAで有意差あり。
- セグメント別で順位変動があります。<br>
→ 理解はできるが 「だから何なの？」 になる。

### ナラティブの役割

| 目的                | 説明                |
| ----------------- | ----------------- |
| 事実を整理する           | ロジカルで客観的な分析       |
| **意味を与える（ナラティブ）** | なぜ起きたのか、何をすべきかを語る |
| 相手を動かす            | 合意形成、投資判断、改善実行    |

#### 重要ポイント

- ナラティブは「感情的な話」ではなく、データを文脈でつなぎ、意思決定の方向性を示すものです。

### ナラティブの例
```
今回のQBR評価では、A社とB社が他社を大きく引き離し、競争関係にあることが明確となりました。
総合スコアでは僅差でA社がトップとなりましたが、その背景には評価者セグメントによる
評価構造の違いが存在しています。

A社は業務部門およびヘビーユーザーから高い評価を獲得しており、特にパフォーマンス面で
強力な強みを持っています。日々の運用品質に対する信頼が厚く、業務インパクトへの理解や
コミュニケーション能力が評価されています。一方で、改善提案に関するスコアが相対的に低く、
プロアクティブな改革・将来視点の提案力には改善余地があります。

対照的に、B社は技術領域で最も高い支持を得ており、課題解決力や専門性に関しては
他社を圧倒しています。しかし、業務視点やビジネス理解に関する評価は低く、
特に業務部門からの評価が伸び悩んでいる点が競争上の課題です。

この構造から、両社は「現在の安定力（A社）」と「技術的成長ポテンシャル（B社）」という
補完関係にあります。A社が改善提案を強化し、B社がビジネス視点を補足することができれば、
どちらもトップポジションを狙える状況です。

今回の分析は単なる順位比較ではなく、投資重点配分の方向性を示しています。
A社には改革力向上への投資、B社には業務理解支援と対話の場づくりを行うことで、
双方の価値がさらに高まる可能性が高いと考えられます。
```
