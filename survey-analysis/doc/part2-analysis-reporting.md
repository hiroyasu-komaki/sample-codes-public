# 総合満足度の算出方法

## 1. 直接測定法(推奨)

### 単一質問による総合満足度
```
質問8: 「IT部門への総合満足度」
□非常に満足(5) □満足(4) □普通(3) □不満(2) □非常に不満(1)
```

**算出方法**
- 各回答者の回答値をそのまま使用
- 全体平均 = Σ(各回答者の評価値) ÷ 回答者数
- 例:平均3.2点(5点満点)

**メリット**
- 回答者の直感的な総合判断
- 算出が簡単
- 他の項目との独立性

<br>

## 2. 複合指標法

### 全項目の単純平均
```
総合満足度 = (対応スピード + 技術的解決力 + 説明のわかりやすさ + 
            対応の丁寧さ + システム安定性 + セキュリティ + 新システム支援) ÷ 7
```

**算出例**
- 対応スピード:3.1
- 技術的解決力:3.4
- 説明のわかりやすさ:2.9
- 対応の丁寧さ:3.3
- システム安定性:3.0
- セキュリティ:3.5
- 新システム支援:3.2
- **総合満足度:3.20**

<br>

## 3. 重要度加重平均法(最も精緻)

### 重要度による重み付け
```
総合満足度 = Σ(各項目の満足度 × 重要度) ÷ Σ(重要度)
```

**算出例**
| 項目 | 満足度 | 重要度 | 重み付け値 |
|------|--------|--------|------------|
| 対応スピード | 3.1 | 4.2 | 13.02 |
| 技術的解決力 | 3.4 | 4.0 | 13.60 |
| 説明のわかりやすさ | 2.9 | 3.8 | 11.02 |
| 対応の丁寧さ | 3.3 | 3.5 | 11.55 |
| システム安定性 | 3.0 | 4.1 | 12.30 |
| セキュリティ | 3.5 | 3.7 | 12.95 |
| 新システム支援 | 3.2 | 3.4 | 10.88 |

**計算**
- 重み付け値合計:85.32
- 重要度合計:26.7
- **総合満足度:85.32 ÷ 26.7 = 3.20**

<br>

## 4. 実践的な推奨アプローチ

### 組み合わせ方式
```
1. 主指標:直接測定の総合満足度(質問8)
2. 副指標:重要度加重平均による算出値
3. 参考指標:単純平均による算出値
```

### 報告書での表現例
```

  - 直接評価:3.2点
  - 重要度加重平均:3.2点
  - 単純平均:3.2点
  
■ 満足度分布
  - 満足以上(4点以上):35%
  - 普通(3点):40%
  - 不満以下(2点以下):25%
```

### データ処理

# IPA分析(重要度・満足度分析)

## 1. IPA分析とは

### Importance-Performance Analysis
```
目的:
改善の優先順位を明確化

方法:
重要度(縦軸)×満足度(横軸)の
マトリックス分析
```

<br>

## 2. 4象限の解釈

### マトリックス構成
```
        重要度高
            ↑
    Ⅱ維持エリア | Ⅰ最優先改善
    -------------|-------------
    Ⅲ改善検討   | Ⅳ過剰品質
            ←
        重要度低
```

### 各象限の意味

**第Ⅰ象限:最優先改善エリア**
```
重要度:高
満足度:低
対応:最優先で改善が必要
```

**第Ⅱ象限:維持エリア**
```
重要度:高
満足度:高
対応:現状を維持・強化
```

**第Ⅲ象限:改善検討エリア**
```
重要度:低
満足度:低
対応:余裕があれば改善
```

**第Ⅳ象限:過剰品質エリア**
```
重要度:低
満足度:高
対応:リソースの再配分を検討
```

<br>
