# IT部門の社内向けサービス満足度アンケート
# Part 1: アンケート設計と前処理

## TL;DR

### アンケートで最も重要なこと
- 母集団サイズよりサンプル数(絶対数)が重要。回答率は統計的精度には影響しない
- ただし代表性(回答の偏り)には影響するので適切なサンプリングが必要

#### 統計的根拠
```
- 必要サンプル数: 357名
- 推奨サンプル数: 600名(安全率68%の余裕)
- 母集団サイズの影響: 5,000人でも100万人でも必要数はほぼ同じ
```

### 信頼性の二つの側面
1. **統計的精度**: 有効回答数で決まる
2. **代表性**: 回答者偏向の有無

### 複数の分析手法を組み合わせる
- **基本分析**: 記述統計・満足度分布
- **詳細分析**: 相関分析・属性別分析・IPA分析

### データ品質確保のために
- **欠損値処理**: 欠損率5%未満→削除、5-15%→平均値補完
- **代表性確認**: 回答者属性が母集団と±5%以内の乖離

#### 欠損値の処理方針
```
1. 欠損率 < 5%:リストワイズ削除
2. 欠損率 5-15%:平均値補完 または 回帰補完
3. 欠損率 15-30%:多重補完法
4. 欠損率 > 30%:調査項目の見直し
```

## 3. アンケート作成時の注意点

### 質問の順序
1. 属性情報(回答しやすい項目から)
2. 利用状況(回答者の経験を思い起こさせる)
3. 満足度評価(核心的な質問)
4. 重要度(重みづけや改善アクションで使用)
5. 改善要望(探索目的)

### 回答選択肢の設計
- 5段階評価で統一(統計分析しやすい)
- 中立的選択肢(「普通」)を含める
- 「該当なし」「わからない」選択肢も考慮

### 質問文の工夫
- 曖昧な表現を避ける
- 誘導的な質問を排除
- 回答者の経験に基づく質問にする

# 欠損値の処理

## 1. 欠損値の種類と原因

### 欠損パターンの分類
```
1. 完全ランダム欠損(MCAR)
   - 回答者が偶然回答し忘れた
   - 処理が比較的簡単

2. ランダム欠損(MAR)
   - 他の回答項目と関連して欠損
   - 例:ITスキル低い人が技術的な質問をスキップ

3. 非ランダム欠損(MNAR)
   - 欠損自体に意味がある
   - 例:不満が高い人が途中で回答を止める
```

### 欠損の原因例
```
- 質問項目の理解困難
- 該当しない状況(例:新システム未利用)
- 回答時間不足
- 意図的な回答拒否
- システムエラー
```

<br>

## 2. 欠損値の処理方法

### A. リストワイズ削除(完全ケース分析)
```
条件:1つでも欠損があるケースを除外
適用場面:欠損率が5%以下の場合

例:
回答者A:満足度[3, 4, 欠損, 3, 4] → 除外
回答者B:満足度[3, 4, 2, 3, 4] → 採用
```

**メリット**
- 分析が単純
- 結果の解釈が明確

**デメリット**
- サンプルサイズの減少
- バイアスの可能性

### B. ペアワイズ削除
```
条件:分析に使用する項目のみ欠損チェック
適用場面:項目間の相関分析など

例:
「対応スピード」と「技術力」の相関分析
→ この2項目に欠損がないケースのみ使用
```

### C. 平均値補完
```
計算方法:
欠損値 = その項目の全回答者平均値

例:
「対応スピード」の平均が3.2の場合
欠損値 → 3.2で補完
```

**適用条件**
- 欠損率10%以下
- 欠損がランダムな場合

### D. 回帰補完
```
計算方法:
他の項目から欠損項目を予測

例:
技術力満足度(欠損) = 0.3×対応スピード + 0.4×説明力 + 0.2×丁寧さ
```

### E. 多重補完法(MI)
```
手順:
1. 欠損値を複数回(通常5-10回)推定
2. 各推定値で分析実行
3. 結果を統合
```

**最も統計的に適切だが、複雑**

<br>

## 3. 実践的な処理方針

### 欠損率による判断基準

**欠損率 < 5%**
```
処理方針:リストワイズ削除
理由:影響が軽微、分析が単純
```

**欠損率 5-15%**
```
処理方針:平均値補完 または 回帰補完
理由:サンプルサイズを維持、精度も確保
```

**欠損率 15-30%**
```
処理方針:多重補完法
理由:バイアス回避が重要
```

**欠損率 > 30%**
```
処理方針:調査項目の見直し
理由:データの信頼性が低下
```

### 項目別の処理方針

**基本属性(部署、職位など)**
```
処理方針:リストワイズ削除
理由:分析の基礎となる重要項目
```

**満足度評価項目**
```
処理方針:平均値補完(欠損率10%以下の場合)
理由:主要な分析対象
```

**自由記述**
```
処理方針:欠損のまま
理由:「無回答」自体が意味を持つ
```

<br>

## 4. 処理の実施例

### ケーススタディ:満足度調査データ

**元データの状況**
```
回答者数:630名
欠損値の状況:
- 総合満足度:10件(1.6%)
- 対応スピード:25件(4.0%)
- 技術的解決力:45件(7.1%)
- ITスキル属性:8件(1.3%)
```

**処理方針の決定**
```
1. ITスキル属性(1.3%):リストワイズ削除
   → 8名を除外、残り622名

2. 総合満足度(1.6%):リストワイズ削除
   → さらに10名除外、残り612名

3. 対応スピード(4.0%):リストワイズ削除
   → さらに23名除外、残り589名

4. 技術的解決力(7.1%):平均値補完
   → 平均3.4点で42件を補完
```

**最終的なサンプル**
```
有効サンプル数:589名
除外数:41名(6.5%)
分析可能性:目標600名に対して589名で十分
```
