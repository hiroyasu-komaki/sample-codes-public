# プロジェクトポートフォリオ需給バランス分析レポート

## 📊 エグゼクティブサマリー

分析期間：**2024年1月〜2028年8月**（約56ヶ月間）

### 主要な発見
- ⚠️ **Technical Capabilityで供給不足が発生**（2024年〜2026年初頭）
- ✅ **Business、Delivery、Leadershipは全期間で供給過剰**
- 🎯 **需要ピークは2025年中盤**（全ケイパビリティで最高水準）
- 📉 **2026年後半以降、需要が急激に減少**

---

## 1️⃣ ケイパビリティ別の需給状況

### 1.1 Business Capability（ビジネスケイパビリティ）

**グラフから読み取れる特徴：**
- 需要ピーク：2025年中盤（約2.5人月）
- 供給：2024年初頭の約1.5人月から段階的に増加し、2026年に約4人月でプラトー
- 需給バランス：**全期間で供給過剰**

**📌 主要な観察ポイント：**

| 期間 | 需要トレンド | 供給トレンド | バランス |
|------|------------|------------|---------|
| 2024年初〜中盤 | 急増（0 → 2.5人月） | 緩やかに増加 | 供給過剰（約1人月の余剰） |
| 2024年後半〜2025年中盤 | 高水準で推移（2〜2.5人月） | 継続増加（3〜3.5人月） | 供給過剰（約1人月の余剰） |
| 2025年後半〜2026年 | 緩やかに減少（2 → 1人月） | ピーク到達（約4人月） | 供給過剰（約3人月の余剰） |
| 2027年〜2028年 | 低水準（0.5人月以下） | 安定（約3.5〜4人月） | 大幅な供給過剰 |

**📊 示唆：**
- ビジネスケイパビリティは**常に余剰状態**
- 需要減少後の2027年以降は**極めて大きな供給過剰**（需要の8倍以上）
- 供給は需要ピーク後も高水準を維持しており、**供給調整が行われていない**

### 1.2 Delivery Capability（デリバリケイパビリティ）

**グラフから読み取れる特徴：**
- 需要ピーク：2025年中盤（約2.5人月）
- 供給：2024年の約2人月から段階的に増加し、2026年に約5人月でプラトー
- 需給バランス：**全期間で供給過剰**

**📌 主要な観察ポイント：**

| 期間 | 需要トレンド | 供給トレンド | バランス |
|------|------------|------------|---------|
| 2024年初〜中盤 | 急増（0 → 2.5人月） | 緩やかに増加 | 供給過剰（約1〜2人月） |
| 2024年後半〜2025年 | 高水準で推移（2〜2.5人月） | 継続増加（3.5〜4人月） | 供給過剰（約2人月） |
| 2026年 | 減少傾向（2 → 1人月） | ピーク到達（約5人月） | 供給過剰（約4人月） |
| 2027年〜2028年 | 低水準（0.5人月以下） | 安定（約5人月） | 大幅な供給過剰 |

**📊 示唆：**
- Deliveryも**Businessと同様のパターン**を示す
- 供給は需要の2倍以上を維持
- 2027年以降の供給過剰が**最も深刻**（需要の10倍以上）

### 1.3 Technical Capability（テクニカルケイパビリティ）

**グラフから読み取れる特徴：**
- 需要ピーク：2025年前半（約5人月）
- 供給：2024年の約1.5人月から緩やかに増加、2026年後半に約4人月でプラトー
- 需給バランス：**2024年〜2026年初頭に供給不足が発生**（赤い影の領域）

**📌 主要な観察ポイント：**

| 期間 | 需要トレンド | 供給トレンド | バランス |
|------|------------|------------|---------|
| 2024年初〜中盤 | 急増（0.5 → 4人月） | 緩やかに増加（1.5 → 2.5人月） | **供給不足**（約-1〜-1.5人月） |
| 2024年後半〜2025年前半 | ピーク期（4〜5人月） | 継続増加（2.5 → 3人月） | **供給不足**（約-1.5〜-2人月） |
| 2025年中盤〜2026年初 | 高水準維持（3.5〜4.5人月） | 継続増加（3 → 3.5人月） | ほぼ均衡〜わずかな不足 |
| 2026年中盤以降 | 急激に減少（3 → 0.5人月） | プラトー（約4人月） | 供給過剰に転換 |
| 2027年〜2028年 | 極めて低水準（0.5人月以下） | 安定（約4人月） | 大幅な供給過剰 |

**📊 示唆：**
- **唯一供給不足が発生したケイパビリティ**
- 不足期間：約24ヶ月（2024年初〜2026年初）
- 最大不足：約-2人月（2025年前半）
- 需要が急減する2026年後半以降、供給過剰に転換
- **需要変動に対する供給の追従が遅れている**

### 1.4 Leadership Capability（リーダーシップケイパビリティ）

**グラフから読み取れる特徴：**
- 需要：全期間で極めて低水準（最大0.8人月程度）
- 供給：2024年の約1.8人月から段階的に増加し、2026年に約4.5人月でプラトー
- 需給バランス：**全期間で大幅な供給過剰**

**📌 主要な観察ポイント：**

| 期間 | 需要トレンド | 供給トレンド | バランス |
|------|------------|------------|---------|
| 2024年初〜中盤 | わずかに増加（0 → 0.5人月） | 増加（1.8 → 3人月） | 大幅な供給過剰（約2.5人月） |
| 2024年後半〜2025年 | 低水準で推移（0.5〜0.8人月） | 継続増加（3 → 3.5人月） | 大幅な供給過剰（約3人月） |
| 2026年〜 | ピーク到達（約0.8人月） | ピーク（約4.5人月） | 大幅な供給過剰（約4人月） |
| 2027年〜2028年 | 減少（0.5 → 0人月） | 安定（約4.5人月） | 極めて大きな供給過剰 |

**📊 示唆：**
- **最も需要が小さい**ケイパビリティ
- 供給は需要の**5〜10倍以上**で推移
- 全期間を通じて**最も非効率な配置**
- 需要に対して供給が**構造的に過剰**

## 3️⃣ 重要な発見と示唆

### ✅ ポジティブな発見

1. **供給不足は限定的**
   - Technical Capabilityのみ、約24ヶ月間の不足
   - 最大不足も-2人月程度と管理可能なレベル
   - 他3つのケイパビリティは常に供給過剰で安全マージン確保

2. **需要ピークが明確**
   - 2025年中盤に集中
   - ピーク期間は約12ヶ月と比較的短期
   - 予測可能な需要変動パターン

3. **供給の段階的増加**
   - 急激な変動なく、計画的な増員が実施されている
   - 供給曲線が滑らか

### ⚠️ 深刻な課題

1. **Technical Capabilityの構造的不足**
   ```
   問題の本質：
   - 需要急増に供給増加が追いつかない
   - 不足期間：24ヶ月（2024年初〜2026年初）
   - 最大ギャップ：約-2人月（実際のプロジェクト遂行に支障）
   
   影響：
   - プロジェクト遅延リスク
   - 品質低下の可能性
   - 既存メンバーの過重負担
   ```

2. **需要減少後の供給過剰**
   ```
   問題の本質：
   - 2026年後半以降、需要が急減するが供給は高止まり
   - 全ケイパビリティで需要の5〜10倍以上の供給
   
   影響：
   - 人件費の無駄
   - 稼働率の低下
   - モチベーション低下リスク
   ```

3. **Leadership Capabilityの極端な過剰**
   ```
   問題の本質：
   - 全期間で需要の5〜10倍以上の供給
   - 需要が最大でも0.8人月程度と極めて小さい
   
   影響：
   - 最も非効率なリソース配置
   - 高コスト人材の遊休化
   ```

4. **需給調整メカニズムの不在**
   ```
   問題の本質：
   - 需要減少に対して供給が全く調整されない
   - 2027〜2028年の需要激減期でも供給は高水準維持
   
   影響：
   - 構造的な非効率性
   - 中長期的なコスト負担
   ```

## 6️⃣ 戦略的推奨事項

### 🚨 緊急対応（即時〜3ヶ月）

#### 1. Technical不足の即時解消

**Priority: 🔴 CRITICAL**

```
アクション:
□ 外部リソースの緊急調達
  - フリーランス/業務委託の活用（目標：+2人月分）
  - 期間：2026年1月まで
  
□ 社内スキル転換プログラムの開始
  - Business/Delivery → Technical への短期トレーニング
  - 対象：希望者3〜5名
  - 期間：3ヶ月集中プログラム
  
□ プロジェクト優先順位の見直し
  - Technical需要の高い案件を優先
  - 並行案件数の制限検討
```

**期待効果：**
- Technical不足の即時解消（-2人月 → 0人月）
- プロジェクト遅延リスクの低減

**Priority: 🔴 HIGH**

```
アクション:
□ 週次の需給レビュー会議の開催

□ リアルタイムダッシュボードの作成
  - 各ケイパビリティの需給状況を可視化
  - アラート設定（不足閾値：-0.5人月）
```

## 📎 補足資料

### 用語定義

- **FTE (Full Time Equivalent)**: フルタイム換算。1人のフルタイム従業員を1.0とする単位
- **人月**: 1人が1ヶ月働いた場合の作業量
- **供給不足**: 需要が供給を上回る状態（グラフの赤い領域）
- **供給過剰**: 供給が需要を上回る状態（グラフの青い領域）

### データの前提条件

1. FTE正規化済み（各社員のケイパビリティ合計を1.0に調整）
2. プロジェクト期間中の需要を均等配分
3. 供給データは実際の配置計画に基づく
4. 分析期間: 2024年1月〜2028年8月

### 分析の制約事項

- 実際の稼働率は考慮していない
- ケイパビリティ内の専門性の違いは反映していない
- 外部要因（市場環境変化等）は想定していない
