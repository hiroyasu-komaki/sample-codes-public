# デジタルスキル標準システム

## 概要

デジタルスキル標準システム - スキルデータを管理・処理するPythonアプリケーションです。

## 必要要件

### システム要件
- Python 3.7以上

### 必須Pythonパッケージ
```
pandas>=1.3.0
PyYAML>=5.4.1
numpy>=1.21.0
```

## インストール

### 仮想環境の作成（推奨）
```bash
python3 -m venv venv

# Mac OSの場合: 
source venv/bin/activate

# Windowsの場合: 
venv\Scripts\activate
```

### 依存パッケージのインストール
```bash
pip install -r requirements.txt
```

## 🏗️ システム構造

```
プロジェクト/
├── main.py                          # メインエントリポイント
├── config/
│   ├── skillstd.yaml               # スキル定義データ
│   └── master_data.yaml            # マスターデータ（職種・レベル定義）
├── modules/
│   ├── skill_standard_processor.py # スキルデータ処理
│   └── skill_consolidator.py       # データ統合処理
├── output/                         # 分割CSVファイル出力先
└── output2/                        # 統合CSVファイル出力先
```

## 🔧 主要コンポーネント

### 1. **main.py** - メインプログラム
インタラクティブメニューで3つの実行モードを提供:

- **Mode 1**: データ生成（分割CSV作成）
- **Mode 2**: データ統合（CSV結合）
- **両方実行**: 生成→統合の順に実行

**主要機能:**
- `create_skill_standard_csv_split()`: ロール×専門性でCSVを分割生成
- `run_data_generation()`: データ生成処理の実行
- `run_data_consolidation()`: データ統合処理の実行

### 2. **skill_standard_processor.py** - データ処理エンジン

**SkillStandardProcessorクラス:**
- YAMLファイル（スキルデータ、マスターデータ）を読み込み
- スキル×ロールの全組み合わせでDataFrameを生成
- データ検証機能を提供

**出力データ構造:**
```
カテゴリー | サブカテゴリー | スキル項目 | ロール | 専門性 | スキルレベル | スキルレベル_数値
```

### 3. **skill_consolidator.py** - データ統合ツール

**SkillConsolidatorクラス:**
- output/ディレクトリ内の全CSVファイルを読み込み
- 1つの統合CSVファイルに結合
- ファイル名カラムを追加してトレーサビリティを確保

### 4. **master_data.yaml** - マスターデータ定義

**定義内容:**
- **11の職種 (roles)**:
  - ビジネス系: 新規事業開発、既存事業の高度化、社内業務の高度化・効率化
  - デザイン系: サービスデザイナー、UX/UIデザイナー
  - データ系: データビジネス、データアナリティクス、データエンジニア
  - エンジニア系: フロントエンド、バックエンド、クラウド/SRE

- **4つの専門性分類 (specialities)**:
  - ビジネスアーキテクト
  - デザイナー
  - データサイエンティスト
  - ソフトウェアエンジニア

- **スキルレベル定義**:
  - `a` (5): 高い実践力と専門性が必要
  - `b` (3): 一定の実践力と専門性が必要
  - `c` (2): 説明可能なレベルで理解が必要
  - `d` (1): 位置づけや関連性の理解が必要
  - `z` (2.5): 役割や状況に応じた実践力が必要

## 🔄 データフロー

```
1. YAMLファイル読み込み
   ↓
2. スキル×ロールのマトリクス展開
   ↓
3. ロール×専門性でグループ化
   ↓
4. 分割CSVファイル生成 (output/)
   ↓
5. 全CSVファイルを統合
   ↓
6. 統合CSVファイル出力 (output2/)
```

## 💡 主な特徴

1. **モジュール設計**: 処理と統合を分離した保守性の高い構造
2. **データ検証**: 必須カラム、NULL値、スキルレベル値の妥当性チェック
3. **詳細ログ**: 絵文字付きの見やすい進捗表示
4. **柔軟な実行**: 必要な処理だけを選択実行可能
5. **トレーサビリティ**: 統合データに元ファイル名を記録

**統合CSVファイル:**
