# ジョブ定義フォーマット：レコードマネジャー職（エキスパートレベル）

## 1. 基本情報
| 項目 | 内容 |
|------|------|
| ジョブタイトル | レコードマネジャー（記録管理専門職） |
| 所属組織 | 情報管理部門 / コンプライアンス部門 / IT部門（記録管理推進組織） |
| 職位レベル | エキスパートレベル |
| 雇用形態 | 正社員（ジョブ型） |

---

## 2. ジョブの目的（Job Purpose）
記録管理の国際標準（ISO15489等）に基づき、  
組織全体の記録の真正性・信頼性・完全性・利用性を保証する  
記録管理体制を構築・運用することを目的とする。

特に、紙から電子記録への移行期における  
記録のライフサイクル管理・メタデータ管理 ・長期保存・リスク管理を高度化し、  
説明責任・コンプライアンス・業務継続における組織の信頼性を高める。

将来的には、Records Management Office（RMO）の中核人材として  
全社的記録管理基盤を推進する役割を担う。

---

## 3. 主な職務内容（Key Responsibilities）

### ● 記録管理戦略・ガバナンス構築
- 記録管理戦略・ロードマップ策定  
- 記録管理方針・規程・手順の策定・更新  
- 記録管理ガバナンス体制（責任者・委員会等）の構築  
- 記録管理成熟度評価および改善計画の策定  

### ● 業務分析・分類体系（BCS）構築
- 業務活動の体系的分析および記録要求事項の特定  
- 業務分類体系（Business Classification Scheme）の設計  
- 保存期間表（Retention Schedule）の策定・維持管理  
- 記録の価値評価（証拠価値・情報価値・歴史的価値）  

### ● メタデータ設計・デジタルアーカイブ基盤構築
- メタデータ枠組み（Metadata Schema）の設計・標準化  
- 電子記録の真正性・完全性確保（署名・タイムスタンプ等）  
- 長期保存戦略・マイグレーション計画の策定  
- ハイブリッド環境（紙・電子）の統合管理  

### ● 記録ライフサイクル管理（Create → Capture → Use → Store → Dispose）
- 記録作成ルール策定と業務プロセスへの組込み  
- 記録の分類・索引付け・保存・利用・廃棄の実務統括  
- 処分権限（Disposition Authority）管理  
- 重要記録（Vital Records）の特別管理  

### ● 記録システム（RMS/ECM）の設計・運用
- 記録管理要件に基づくシステム要件定義  
- 記録システム（RMS/ECM/DAM等）の設計・導入リード  
- メタデータ実装・検索性向上・アクセス制御の設計  
- システム監査・品質評価・改善  

### ● コンプライアンス・監査・訴訟対応
- 法規制（公文書管理法、情報公開法、個人情報保護法等）への準拠  
- 内部・外部監査対応、記録管理成熟度評価  
- 訴訟におけるLegal Hold、eディスカバリー支援  
- 情報漏洩防止・改ざん防止の統制強化  

### ● 教育・文化醸成
- 記録管理研修（新入社員・管理職・専門研修）企画・実施  
- マニュアル・ガイドラインの整備、eラーニング教材の開発  
- 組織全体の記録管理意識向上（啓発活動・ベストプラクティス共有）  
- 継続的改善と文化醸成の推進（PDCA）  

---

## 4. 成果基準（Expected Outcomes / KPIs）
- 記録管理成熟度（Maturity Model）の向上  
- 記録の真正性・完全性・利用性に関する品質指標の改善  
- 監査対応・情報公開請求対応の迅速性・正確性  
- 保存期間遵守率・不適切廃棄ゼロの達成  
- 記録システムのユーザビリティ向上・利用率増加  
- 組織横断の記録管理標準化・整合性の向上  

---

## 5. 必要スキル・知識（Required Skills / Knowledge）
| カテゴリ | 内容 |
|-----------|------|
| 記録管理専門知識 | ISO15489、ISO30300、メタデータ、分類体系、Retention管理 |
| デジタル記録管理 | 電子記録、デジタルアーカイブ、電子署名、タイムスタンプ |
| 業務分析 | 業務プロセス分析、記録要求事項分析、価値評価 |
| システム | RMS/ECM、メタデータ管理ツール、SharePoint等 |
| 法規制 | 公文書管理法、個人情報保護法、情報公開法等 |
| コミュニケーション | 経営層説明力、調整力、教育・啓発スキル |
| リスク管理 | 記録管理リスク評価、Legal Hold、監査対応 |

---

## 6. 必要経験（Required Experience）
- 記録管理または文書管理実務経験 5年以上  
- 中規模以上の記録管理システム導入・運用プロジェクト経験  
- 記録ライフサイクル管理（3年以上）  
- 業務分類体系（BCS）・保存期間表の設計経験  
- 訴訟・監査・情報公開対応の実務経験（望ましい）  
- グローバル記録管理標準化の参画経験（望ましい）  

---

## 7. 権限と責任範囲（Authority & Accountability）
- 記録管理方針・基準・保存期間の決定権  
- 記録管理ガバナンスの推進・監査対応責任  
- 記録システム要件定義・ベンダー評価の主導  
- 記録の作成・保存・処分に関する全社統制責任  
- Legal Hold 等の訴訟対応における記録管理責任
