# 募集要項

## 募集職種

### システムアーキテクト/ソリューションアーキテクト職(エキスパートレベル)

<br>

## 現状の課題

### システム・ソリューションアーキテクチャ設計における専門能力の不足
- 既存のIT戦略実行を進める中で、ビジネス要件を最適な技術ソリューションに変換する専門能力や深い知見の不足が課題
- 現場においても個別システムの設計は実施しているが、エンタープライズアーキテクチャとの整合性を保ちながら、技術的に優れたソリューションを設計するスピード感や実行精度において専門性の補完が必要な状況
- 特に、システム全体アーキテクチャの設計、技術選定・評価、非機能要件(性能・拡張性・セキュリティ)の実現、システム間統合設計、クラウドネイティブアーキテクチャの適用など、技術的視点からのアーキテクチャ設計において、戦略目標の達成に向け現状の実行力は十分ではない

### IT部門グローバル化における技術統合力不足
- 海外組織との連携強化、グローバル共通の技術標準・アーキテクチャパターンの構築、統一された技術選定基準の確立が必要
- 複数地域・組織にまたがるシステムアーキテクチャの統合設計、効果的な技術ガバナンス体制など、グローバルレベルでの技術統合力強化

### 全社横断的技術アーキテクチャ実行力の強化
- IT部門単独ではなく、経営企画部門、事業部門、外部パートナーなどと連携し、システムアーキテクチャのプロセス標準を通して各部門との協働による実行力強化
- 将来的には、テクニカルアーキテクチャ推進組織を組織化することもあるが、そのための試金石として、その中核人材が必要

<br>

## 期待する役割・貢献

### システム・ソリューションアーキテクチャ設計力の直接強化
- 当社の戦略実行において不足している専門能力を直接補完し、システムアーキテクチャ設計を通じたビジネス価値実現と技術的優位性の実行力を強化していただきます
- 現場の取り組みに専門的な技術視座とスピード感を付加し、システム開発・導入プロジェクトの成功確度を高めていただきます。

### グローバル技術アーキテクチャ基盤の強化
- 統一された技術標準・アーキテクチャパターンの確立、海外組織との効果的な技術協働体制構築など、グローバル技術アーキテクチャ基盤の強化をリードしていただきます。

### 組織技術アーキテクチャ能力の向上
- システムアーキテクチャに関する専門知識・ノウハウの社内展開、技術標準プロセスの整備、エンジニアのスキル向上支援など、組織全体の技術アーキテクチャ能力向上に貢献していただきます。

<br>

## 必須要件

### 実務経験
- システムアーキテクチャ設計・ソリューションアーキテクチャ設計業務経験5年以上
- 大規模システム開発プロジェクトにおけるアーキテクト役割での実務経験(中規模以上のプロジェクト3件以上)
- システム統合・技術選定・非機能要件設計の実務経験3年以上

### 専門知識・スキル
- システムアーキテクチャ設計・ソリューション設計
- 技術選定・評価(プログラミング言語、フレームワーク、ミドルウェア等)
- 非機能要件設計(性能、可用性、拡張性、セキュリティ等)
- システム統合設計・インターフェース設計
- クラウドアーキテクチャ(AWS、Azure、GCP等)
- マイクロサービスアーキテクチャ・API設計
- データアーキテクチャ・データベース設計
- セキュリティアーキテクチャ設計
- DevOps・CI/CDアーキテクチャ
- エンタープライズアーキテクチャ(EA)に関する基礎知識
- 企業IT・システム管理に関する基礎知識
- ビジネスプロセス・業務要件に関する基礎知識
- 経営企画・統制に関する基礎知識
- リスクマネジメント
  
### コミュニケーション能力
- 社内外のステークホルダーとの調整・技術説明能力
- 経営層への技術戦略報告・提案スキル
- 開発チーム・ベンダーとの連携・技術指導経験

### 語学力
- 日本語ネイティブレベル(日本語検定1級または同等レベル)
- 英語フルエントレベル(TOEIC 900点以上またはCEFR C1レベル以上)
- ビジネス実務と技術実務のそれぞれのシーンにおいて、相手に合わせて専門用語と平易な表現を使い分けられる程度に両言語を習熟

### システム・ツール
- Microsoft Office系ビジネスツールに関する習熟(特にExcel上級レベル、Visio等のモデリングツール、PowerBI等の可視化ツール使用経験必須)
- Microsoft系コラボレーションツールに関する習熟(SharePoint、Teams、OneNote、Planner等のチーム作業ツール群)
- アーキテクチャ設計ツール(Enterprise Architect、Lucidchart、draw.io等)の利用経験、かつツール導入における要件定義フェーズへの参加経験(アーキテクチャ管理要件整理・必要機能の検討経験)

### その他
- システムアーキテクチャ・ソリューション設計分野における包括的なプロフェッショナリズム(専門性、職業倫理、責任感、継続的学習への取り組み)
- システムアーキテクチャを通して企業に貢献し変革したいという意思、および業界や社会に対してポジティブに取り組む意欲
- 上記目標実現のための継続的な自己研鑽・情報収集・知識更新への具体的な取り組みを実践

# 業務レベル要件定義

## 業務レベルの評価軸

### 影響範囲による評価
- **組織**: 所属組織内 / 部門横断 / 全社的 / グループ会社含む
- **地域**: 国内のみ / 国内中心 / 国内外含む / グローバル統括

### 難易度による評価
- **業務複雑性**: 定型業務 / 半定型業務 / 非定型業務 / 創造的業務
- **意思決定レベル**: 実行レベル / 戦術レベル / 戦略レベル / 経営レベル
- **専門性要求度**: 基礎知識 / 応用知識 / 専門知識 / エキスパート知識

<br>

## レベル定義

| レベル | 組織 | 地域 | 業務複雑性 | 意思決定レベル | 専門性要求度 | 特徴 |
|--------|------|------|-----------|---------------|-------------|------|
| **ジュニアレベル** | 所属組織内 | 国内のみ | 定型業務 | 実行レベル | 基礎知識 | 既定の手順・基準に従った業務実行、上位者の指導下での遂行 |
| **中級レベル** | 所属組織内〜部門横断 | 国内のみ〜国内中心 | 半定型業務 | 実行レベル〜戦術レベル | 基礎知識〜応用知識 | 一定の裁量権を持った業務遂行、チーム指導、改善提案 |
| **エキスパートレベル** | 部門横断〜全社的 | 国内中心〜国内外含む | 非定型業務 | 戦術レベル〜戦略レベル | 専門知識 | 専門領域でのリーダーシップ、制度設計・仕組み構築、重要案件担当 |
| **シニアレベル** | 全社的〜グループ会社含む | 国内外含む〜グローバル統括 | 創造的業務 | 戦略レベル〜経営レベル | エキスパート知識 | 全社・グループレベルでの戦略策定・統括、経営層への直接報告・提案 |
