# README：ベンダーマネジメントガイド

## 概要

このフォルダには、企業IT組織におけるベンダーマネジメント（Vendor Management）に関する包括的なガイド文書が含まれています。VMO（Vendor Management Office / ベンダーマネジメント担当者）が日々の業務で直面する課題に対応するための実践的な手引きとして構成されています。

## 文書間の関係性

### 1. 総合ガイドと実践ガイドの関係

- **ベンダーマネジメントガイド.md**：企業IT調達・ベンダーマネジメントの全体像を4階層のガバナンスレイヤで説明。COBIT 2019との対応も含む包括的なガイド。
- **vmo-guide.md 以降**：VMO（ベンダーマネジメント担当者）の視点から、実務で必要な具体的な手順・チェックリスト・テンプレートを提供。

### 2. 実践ガイドの章構成と依存関係

```
第1章（vmo-guide.md）
  ↓ 基盤となる役割理解
第2章（vmo-guide-procurement.md）
  ↓ 調達プロセスの理解
第3章（vmo-guide-lifecycle-management.md）
  ↓ ライフサイクル管理の実践
第4章（vmo-guide-responsibility.md）
  ↓ 特殊ケースへの対応
第5章（vmo-guide-raci.md）
  ↓ 組織内での連携
第6-7章（vmo-guide-kpi.md）
  ↓ 成果測定とチェックリスト
付録（vmo-guide-appendix.md）
  └─ 実務で使えるテンプレート集
```

## 各文書の概要

### 総合ガイド

#### ベンダーマネジメントガイド.md
- **目的**：企業IT調達・ベンダーマネジメントの全体像を理解する
- **対象読者**：経営層、EA/PMO、VMO、IT購買担当者
- **主な内容**：
  - 4階層のガバナンスレイヤ（戦略・ガバナンス → 調達戦略・ポリシー → 個別案件の調達プロセス → ベンダーライフサイクル管理）
  - 調達プロセスの標準ステップ（8ステップ）
  - 運用開始後のRACIマトリクス
  - COBIT 2019とのプロセス対応

### 実践ガイド（VMO向け）

#### vmo-guide.md（第1章：役割と位置づけ）
- **目的**：VMOの役割と組織内での位置づけを理解する
- **主な内容**：
  - ベンダーマネジメントの責任（ベンダー台帳管理、分類・リスク評価、パフォーマンスレビュー、契約交渉、関係性構築）
  - VMOが関わる組織との関係図
  - COBIT 2019フレームワークとの対応（APO10が中核）

#### vmo-guide-procurement.md（第2章：調達プロセス）
- **目的**：調達プロセス全体でVMOが果たすべき役割を理解する
- **主な内容**：
  - 調達プロセス全体の流れ（シーケンス図）
  - 調達の8ステップとVMOのアクション
  - VMOが作成・管理すべき主要文書

#### vmo-guide-lifecycle-management.md（第3章：ベンダーライフサイクル管理）
- **目的**：契約締結から終了まで、継続的にベンダーを管理する方法を理解する
- **主な内容**：
  - ベンダー台帳の作成と管理（ER図、分類基準）
  - オンボーディング（30日計画のガントチャート）
  - パフォーマンス/リスクモニタリング（月次・四半期レビュー）
  - 関係性・コスト最適化
  - リスク管理とエスカレーション
  - オフボーディング（90日計画のガントチャート）

#### vmo-guide-responsibility.md（第4章：クラウド/SaaS・運用保守委託でのVMOの特別な注意点）
- **目的**：クラウド/SaaS・運用保守委託に特有のリスクと管理ポイントを理解する
- **主な内容**：
  - クラウド/SaaS調達でのVMOの追加業務（データ所在地、ロックイン回避）
  - 運用保守委託でのVMOの管理ポイント（L1-L4サポート体制、ブラックボックス化防止）
  - VMOが交渉すべきSLA設計（可用性、インシデント対応）
  - セキュリティ・データ保護（Shared Responsibility Model）
  - 継続監視ダッシュボード

#### vmo-guide-raci.md（第5章：VMOの組織内での役割と連携）
- **目的**：VMOが組織内でどのように連携すべきかを理解する
- **主な内容**：
  - VMOの組織上の位置づけ（組織図）
  - 調達プロセスでのVMOの役割（RACIマトリクス）
  - 運用フェーズでのVMOの役割（RACIマトリクス）
  - エスカレーションルート
  - VMOが活用すべきツール（契約管理、SLA監視、調達管理、リスク管理）
  - スキル向上ロードマップ

#### vmo-guide-kpi.md（第6章：成果を測る指標（KPI）、第7章：VMOの実践チェックリスト）
- **目的**：VMOの成果を測定し、日常業務を効率化する
- **主な内容**：
  - VMOの個人KPI（効率性、コスト、品質、リスク）
  - VMO組織全体のKPI
  - 成熟度向上（COBIT準拠、Level 1-5）
  - 実践チェックリスト（週次、月次、四半期、年次、緊急時）

#### vmo-guide-appendix.md（付録）
- **目的**：実務で即座に使えるテンプレートとQ&Aを提供する
- **主な内容**：
  - 付録A：テンプレート集（ベンダー台帳、RFP評価マトリクス、月次レビューアジェンダ、ベンダー評価シート、オフボーディングチェックリスト）
  - 付録B：Q&A（VMO設置タイミング、SLA未達対応、価格削減交渉、ロックイン回避、評価が低い場合の判断）

## 読み進め方の推奨

### 初めて読む方（VMO初心者）

1. **vmo-guide.md**（第1章）でVMOの役割と位置づけを理解
2. **vmo-guide-procurement.md**（第2章）で調達プロセスの基本を把握
3. **vmo-guide-lifecycle-management.md**（第3章）でライフサイクル管理の全体像を理解
4. **vmo-guide-appendix.md**（付録）のテンプレートを確認し、実務に活用

### 実務で活用する方（VMO経験者）

1. **vmo-guide-kpi.md**（第6-7章）のチェックリストを日常業務に組み込む
2. **vmo-guide-responsibility.md**（第4章）で特殊ケースへの対応を確認
3. **vmo-guide-raci.md**（第5章）で組織内連携を最適化
4. **vmo-guide-appendix.md**（付録）のテンプレートをカスタマイズして使用
