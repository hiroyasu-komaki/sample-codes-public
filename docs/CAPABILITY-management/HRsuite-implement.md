
## IT人材育成システム

### システム構成

```
【人事コアシステム】
ERP（人事給与モジュール）
├─ 基本人事情報
├─ 給与計算
├─ 勤怠管理
└─ 組織構造

【人事専門システム】
├─ タレントマネジメントシステム
│   ├─ スキル管理
│   ├─ キャリアパス
│   └─ 後継者計画
├─ LMS（学習管理）
│   ├─ 研修プログラム
│   ├─ eラーニング
│   └─ 資格管理
└─ パフォーマンス管理
    ├─ 目標管理
    ├─ 評価
    └─ フィードバック

【その他連携システム】
├─ プロジェクト管理ツール
│   ├─ リソース配置
│   ├─ 稼働管理
│   └─ スキルマッチング
│
└─ ・・・
```

## ERPとの統合パターン

### 統合パターンの種類と特徴

| 項目 | パターン1：ERP統合型 | パターン2：ベストオブブリード型 | パターン3：人事特化スイート型 |
|------|------|------|------|
| **概要** | ERPベンダーの人事モジュールとERPを統合 | 各領域で最適なツールを選択しAPI連携 | 人事領域を包括的にカバーするスイートを導入 |
| **代表的な構成例** | ・SAP ERP + SAP SuccessFactors<br>・Oracle ERP + Oracle HCM Cloud<br>・Microsoft Dynamics 365 + Dynamics 365 HR | ・SAP ERP（給与）<br>・Workday（タレント管理）<br>・Cornerstone（LMS）<br>・Jira（プロジェクト管理）<br>・Culture Amp（エンゲージメント） | ・Workday HCM（統合スイート）<br>・Oracle HCM Cloud（統合スイート）<br>・SAP SuccessFactors（統合スイート） |
| **提供ベンダー** | 単一ベンダー（ERPベンダー） | 複数ベンダー（各専門ベンダー） | 単一ベンダー（人事特化ベンダー） |
| **カバー範囲** | ・ERP全機能<br>・人事給与<br>・タレント管理<br>・学習管理<br>・パフォーマンス管理 | ・ERP全機能（別ベンダー）<br>・人事給与（別ベンダー）<br>・タレント管理（別ベンダー）<br>・LMS（別ベンダー）<br>・その他専門ツール | ・人事給与<br>・タレント管理<br>・学習管理<br>・採用管理<br>・パフォーマンス管理<br>※ERP財務等は別システム |
| **データ統合** | シームレス（同一ベンダー内） | API連携、ETL、iPaaS等で実現 | 人事領域内はシームレス、ERP等とはAPI連携 |
| **導入コスト** | 高額<br>（数千万円〜数億円） | 中〜高額<br>（ツール数により変動、数千万円〜） | 中〜高額<br>（数千万円〜数億円） |
| **導入期間** | 長期<br>（1年〜3年以上） | 中〜長期<br>（6ヶ月〜2年、段階的導入可能） | 中期<br>（6ヶ月〜2年） |

### 評価軸

| 項目 | パターン1：ERP統合型 | パターン2：ベストオブブリード型 | パターン3：人事特化スイート型 |
|------|------|------|------|
| **運用の複雑さ** | 低〜中<br>（単一ベンダー管理） | 高<br>（複数ベンダー管理、統合維持） | 低〜中<br>（人事領域は単一、ERPとの連携のみ） |
| **柔軟性** | 低〜中<br>（ベンダーの機能範囲内） | 非常に高<br>（最適ツールを自由に選択・変更可能） | 中<br>（人事領域内では柔軟、スイート外は連携必要） |
| **カスタマイズ性** | 中<br>（ベンダー標準に準拠） | 高<br>（各ツールで最適化可能） | 中<br>（スイート内で一定の柔軟性） |
| **ベンダーロックイン** | 高<br>（単一ベンダー依存大） | 低<br>（個別ツール入替可能） | 中<br>（人事領域で依存、他領域は独立） |
| **アップデート管理** | 比較的容易<br>（統一スケジュール） | 複雑<br>（各ベンダー個別対応） | 比較的容易<br>（スイート内は統一） |
| **サポート体制** | 単一窓口<br>（ベンダー一社） | 複数窓口<br>（各ベンダー個別） | 単一窓口<br>（人事領域、ERP等は別） |
| **技術スタック** | 統一<br>（同一ベンダー技術） | 多様<br>（各ツールで異なる） | 人事領域で統一<br>（他システムと多様） |
| **データガバナンス** | 強固<br>（単一システム内） | 要注意<br>（データ分散、整合性管理必要） | 人事データは強固<br>（他領域との整合性管理必要） |
| **機能の深さ** | 中<br>（包括的だが各領域は標準的） | 高<br>（各領域でベストツール選択） | 高<br>（人事領域特化で深い機能） |
| **機能の広さ** | 非常に広い<br>（全社業務カバー） | 非常に広い<br>（必要に応じて追加） | 人事領域に限定<br>（財務等は別システム） |
| **導入リスク** | 中<br>（大規模プロジェクト） | 高<br>（統合の複雑さ、プロジェクト管理難） | 中<br>（人事領域は標準化） |
| **将来の拡張性** | 中<br>（ベンダーロードマップ依存） | 高<br>（新ツール追加容易） | 中<br>（人事領域内では拡張容易） |

### ProsCons

## MRP・MRP II・ERPの対比表

| 項目 | MRP<br>（Material Requirements Planning） | MRP II<br>（Manufacturing Resource Planning） | ERP<br>（Enterprise Resource Planning） |
|------|------|------|------|
| **日本語名称** | 資材所要量計画 | 製造資源計画 | 統合基幹業務システム |
| **登場時期** | 1960年代後半〜1970年代 | 1980年代 | 1990年代〜 |
| **管理対象** | 資材・部品 | 製造に関わる全資源（資材、設備、人員、資金） | 企業全体の経営資源（ヒト、モノ、カネ、情報） |
| **適用範囲** | 生産・購買部門 | 製造業の全部門 | 全業種・全部門（製造、販売、財務、人事など） |
| **主な目的** | 資材の適時適量調達 | 製造資源の最適化 | 全社的な経営資源の統合管理と最適化 |
| **対象業種** | 製造業 | 製造業 | 製造業、小売業、サービス業、金融業など全業種 |
| **主要機能** | ・資材所要量計算<br>・発注計画<br>・在庫管理 | ・MRP機能<br>・能力計画<br>・現場管理<br>・購買管理<br>・原価管理 | ・財務会計<br>・管理会計<br>・販売管理<br>・購買管理<br>・生産管理<br>・在庫管理<br>・人事給与<br>・物流管理 |
| **入力情報** | ・基準生産計画（MPS）<br>・部品表（BOM）<br>・在庫情報<br>・リードタイム | ・MRPの入力情報<br>・設備能力情報<br>・人員情報<br>・コスト情報 | ・全部門の業務データ<br>・顧客情報<br>・財務データ<br>・人事データなど |
| **出力情報** | ・発注指示<br>・製造指示 | ・発注計画<br>・製造指示<br>・負荷計画<br>・原価情報<br>・財務予測 | ・各種帳票<br>・経営分析レポート<br>・ダッシュボード<br>・KPI<br>・予算実績対比 |
