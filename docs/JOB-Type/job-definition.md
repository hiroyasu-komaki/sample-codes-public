```
例：

```

日常的に遂行するタスクや成果責任を列挙します。
成果物ベースで書くのがポイントです。
```
例：
事業部門からの依頼に基づくデータ分析およびレポート作成
分析基盤の運用・改善に関する提案と実施
BIツールによるダッシュボード構築
データ品質の維持とメタデータ管理
```

### 4. 成果基準（Expected Outcomes / KPIs）

```

分析提案による意思決定貢献件数

```

### 6. 必要経験（Required Experience）

### 補足：ジョブ定義のレベル設計のコツ
- 1つのジョブは「成果責任と専門性の単位」で区切る。
- 「人」ではなく「仕事（Job）」を記述する。
- 同じタイトルでも、成果責任が異なる場合は別ジョブとして扱う。
