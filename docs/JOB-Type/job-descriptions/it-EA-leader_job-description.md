# 募集要項

## 前提

本募集要項では、TOGAFに基づくエンタープライズアーキテクチャ（EA）を組織に導入する前提で、組織運営に必要な職種・ロールとして、以下のような主要な役割を定義します。現時点でいずれの職種も役割も明文で定義されたものはないため、職種は便宜的に定義したものです。

### 主要な職種と役割
- エンタープライズアーキテクト（Enterprise Architect）：組織全体のビジネス戦略とIT戦略の整合を図り、アーキテクチャ全体の設計・文書化・維持管理を担います。
- ビジネスアーキテクト（Business Architect）：ビジネスプロセスや組織構造を分析し、ビジネス目標達成に向けてEAの観点から設計と調整を行います。
- システムアーキテクト（System Architect） / ソリューションアーキテクト（Solution Architect）：技術的な視点で具体的なシステムやソリューションの設計を行い、EAとの整合性を保ちます。
- EAリーダー（Enterprise Architecture Leader）：EAチームの立ち上げや運営を主導し、EA活動の価値を組織内で示す役割を担います。
- EA補佐：EAチームの補佐またはプロジェクトにおけるEA活動の調整、関係者間の連携や進捗管理を担当します。

## 現状の課題

### エンタープライズアーキテクチャ推進における専門能力の不足
- 既存のIT戦略実行を進める中で、ビジネス戦略とIT戦略の統合的な視点でアーキテクチャを設計・推進する専門能力や深い知見の不足が課題
- 現場においても個別システムのアーキテクチャ設計は実施しているが、エンタープライズ全体視点での戦略的アーキテクチャ構築のスピード感や実行精度において専門性の補完が必要な状況
- 特に、TOGAFに基づくEAフレームワークの確立、ビジネス・データ・アプリケーション・テクノロジーの4層アーキテクチャの統合設計、アーキテクチャガバナンスの体制構築、EA活動の組織的価値創出など、エンタープライズアーキテクチャを通じた全社最適化において、戦略目標の達成に向け現状の実行力は十分ではない

### IT部門グローバル化におけるEA推進力不足
- 海外組織との連携強化、グローバル共通のEAフレームワーク・標準の構築、統一されたアーキテクチャ原則・ガイドラインの確立が必要
- 複数地域・事業にまたがるエンタープライズアーキテクチャの統合設計、効果的なアーキテクチャガバナンス体制など、グローバルレベルでのEA推進力強化

### 全社横断的EA推進体制の強化
- IT部門単独ではなく、経営企画部門、財務部門、各事業部門などと連携し、エンタープライズアーキテクチャのプロセス標準を通して各部門との協働による実行力強化
- 将来的には、EA推進組織(Enterprise Architecture Office)を組織化することもあるが、そのための試金石として、その中核人材が必要

<br>

## 期待する役割・貢献

### EA活動の立ち上げ・推進リーダーシップ
- 当社のEA活動において不足している専門能力を直接補完し、TOGAFに基づくエンタープライズアーキテクチャフレームワークの確立とEAチームの立ち上げ・運営をリードしていただきます
- EA活動の組織的価値を経営層・事業部門に示し、全社的なEA推進の基盤を構築していただきます。

### グローバルEA標準・ガバナンスの確立
- 統一されたEAフレームワーク・アーキテクチャ原則の確立、海外組織との効果的なEA協働体制構築など、グローバルEA推進基盤の強化をリードしていただきます。

### 組織EA能力の向上
- エンタープライズアーキテクチャに関する専門知識・ノウハウの社内展開、EA標準プロセスの整備、アーキテクト人材のスキル向上支援など、組織全体のEA能力向上に貢献していただきます。

# 業務レベル要件定義

## 業務レベルの評価軸

### 影響範囲による評価
- **組織**: 所属組織内 / 部門横断 / 全社的 / グループ会社含む
- **地域**: 国内のみ / 国内中心 / 国内外含む / グローバル統括

### 難易度による評価
- **業務複雑性**: 定型業務 / 半定型業務 / 非定型業務 / 創造的業務
- **意思決定レベル**: 実行レベル / 戦術レベル / 戦略レベル / 経営レベル
- **専門性要求度**: 基礎知識 / 応用知識 / 専門知識 / エキスパート知識
