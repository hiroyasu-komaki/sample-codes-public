## 1. 目的

本文書は、記録の真正性・信頼性・完全性・利用性を確保するために、ISO 30301およびISO 15489に基づくメタデータ要件を定義し、既存のISO 27001情報資産台帳を拡張することを目的とする。

---

## 2. メタデータの定義と目的

### 2.1 メタデータとは

記録の内容、構造、文脈を説明するデータであり、記録の管理と利用を可能にする情報。

### 2.2 メタデータの目的

| 目的 | 説明 | ISO 15489参照 |
|------|------|--------------|
| 真正性の証明 | 作成者・作成日時・承認者の記録 | 4.2.1 |
| 信頼性の確保 | 業務文脈との関連性の明示 | 4.2.2 |
| 完全性の維持 | 変更履歴・バージョン管理 | 4.2.3 |
| 利用性の向上 | 検索・分類・アクセス制御 | 4.2.4 |

---

## 3. メタデータ体系

### 3.1 メタデータの3要素（ISO 15489準拠）

```
メタデータ
├── 記録メタデータ（Record Metadata）：記録そのものの属性
├── 業務メタデータ（Business Metadata）：業務文脈との関連
└── 管理メタデータ（Management Metadata）：記録管理に必要な情報
```

## 4. 統合メタデータ項目一覧

### 4.1 基本識別情報（ISO 27001既存項目）

| 項目名 | 説明 | データ型 | 必須/任意 | ISO 27001での用途 | ISO 30301での追加要件 |
|--------|------|---------|----------|-----------------|---------------------|
| 資産ID | 一意識別子 | 文字列 | 必須 | 情報資産の識別 | 記録IDとして継続使用 |
| 資産名称 | 記録の名称 | 文字列 | 必須 | 資産名 | 記録タイトルとして利用 |
| 分類 | セキュリティ分類 | 選択肢 | 必須 | 機密性レベル | 記録分類と統合 |
| 所有者 | 資産管理責任者 | ユーザーID | 必須 | 情報資産オーナー | 記録管理責任者として継続 |
| 作成日 | 作成日時 | 日時 | 必須 | 資産登録日 | 記録作成日時として利用 |
| 更新日 | 最終更新日時 | 日時 | 必須 | 変更追跡 | 記録変更日時として利用 |

### 4.2 記録管理拡張項目（ISO 30301新規追加）

| 項目名 | 説明 | データ型 | 必須/任意 | 入力例 | 自動/手動 |
|--------|------|---------|----------|--------|----------|
| **記録種別** | 記録の分類 | 選択肢 | 必須 | 契約書、請求書、議事録 | 手動選択 |
| **記録ID** | 記録保持スケジュール参照 | 文字列 | 必須 | REC-L-001 | 自動採番可 |
| **業務プロセス** | 記録が発生した業務 | 選択肢 | 必須 | 契約締結、採用管理 | 自動判定可 |
| **業務文脈** | 記録の背景・目的 | テキスト | 任意 | 2024年度A社との新規取引 | 手動入力 |
| **法的根拠** | 保持根拠となる法令 | 選択肢 | 条件付必須 | 商法522条、労基法109条 | 自動判定 |
| **保持期間** | 保存しなければならない期間 | 数値+単位 | 必須 | 10年、退職後5年 | 自動判定 |
| **起算日** | 保持期間の開始日 | 日付 | 必須 | 2024-01-01、契約終了日 | 手動/自動 |
| **廃棄予定日** | 計算された廃棄可能日 | 日付 | 自動計算 | 2034-01-01 | 自動計算 |
| **最終処分** | 廃棄または永久保存 | 選択肢 | 必須 | 廃棄、永久保存、アーカイブ | 自動判定 |
| **リーガルホールド** | 廃棄保留フラグ | ブール | 必須 | TRUE/FALSE | 手動設定 |

## 5. メタデータスキーマ（データベース設計）

### 5.1 統合資産・記録管理テーブル（拡張版）

```sql
CREATE TABLE integrated_asset_records (
    -- ISO 27001 既存項目
    asset_id VARCHAR(50) PRIMARY KEY,
    asset_name VARCHAR(255) NOT NULL,
    security_classification VARCHAR(20) NOT NULL,
    owner_id VARCHAR(50) NOT NULL,
    created_date TIMESTAMP NOT NULL,
    updated_date TIMESTAMP NOT NULL,
    
    -- ISO 30301 記録管理拡張項目
    record_type VARCHAR(50) NOT NULL,
    record_id VARCHAR(50) NOT NULL,
    business_process VARCHAR(100) NOT NULL,
    business_context TEXT,
    legal_basis VARCHAR(100),
    retention_period VARCHAR(50) NOT NULL,
    retention_start_date DATE NOT NULL,
    disposal_date DATE,
    final_disposition VARCHAR(20) NOT NULL,
    legal_hold BOOLEAN DEFAULT FALSE,
    
    -- 真正性・証拠能力
    creator_id VARCHAR(50) NOT NULL,
    creator_name VARCHAR(100) NOT NULL,
    creation_timestamp TIMESTAMP NOT NULL,
    approver_id VARCHAR(50),
    approval_timestamp TIMESTAMP,
    digital_signature TEXT,
    timestamp_token TEXT,
    hash_value VARCHAR(256),
    
    -- 完全性・変更管理
    version VARCHAR(20) NOT NULL,
    change_log TEXT,
    modifier_id VARCHAR(50),
    modification_timestamp TIMESTAMP,
    change_reason TEXT,
    previous_version_link VARCHAR(100),
    
    -- アクセス・セキュリティ
    access_rights VARCHAR(50) NOT NULL,
    access_scope VARCHAR(50) NOT NULL,
    encryption_method VARCHAR(50),
    storage_location VARCHAR(255) NOT NULL,
    
    -- 利用性・検索
    keywords TEXT,
    tags VARCHAR(255),
    related_records VARCHAR(255),
    language VARCHAR(20),
    file_format VARCHAR(20),
    file_size BIGINT,
    
    -- 関連情報
    counterparty VARCHAR(255),
    project VARCHAR(100),
    department VARCHAR(50) NOT NULL,
    contract_number VARCHAR(50),
    amount DECIMAL(15,2)
```

## 6. メタデータ入力ルール

### 6.1 必須項目の自動入力

以下の項目はシステムが自動で入力する：

| 項目 | 取得方法 |
|------|---------|
| 作成者・作成日時 | ログインユーザー・システム時刻 |
| 変更者・変更日時 | 変更時のユーザー・システム時刻 |
| ハッシュ値 | ファイル保存時にSHA-256で自動生成 |
| 廃棄予定日 | 起算日+保持期間で自動計算 |
| ファイル形式・サイズ | アップロード時に自動判定 |

### 6.2 記録種別による条件分岐

```
IF 記録種別 = "契約書" THEN
    契約番号: 必須
    取引先: 必須
    金額: 必須
    電子署名: 推奨
END IF

IF 記録種別 = "人事記録" THEN
    個人情報フラグ: TRUE
    アクセス権限: 人事部のみ
    暗号化: 必須
END IF
```

## 8. メタデータの利用シナリオ

### 8.1 検索シナリオ

**例1: 特定取引先との契約書を検索**
```
記録種別 = "契約書" AND 取引先 = "株式会社A社"
→ 結果: 該当契約書のリスト（有効期限順）
```

**例2: 廃棄予定の記録を抽出**
```
廃棄予定日 BETWEEN 今日 AND 今日+30日 AND リーガルホールド = FALSE
→ 結果: 30日以内に廃棄可能な記録のリスト
```

### 8.2 コンプライアンス確認

**保持期間違反のチェック**
```
廃棄予定日 < 今日 AND 最終処分 != "永久保存"
→ アラート: 保持期間超過の記録（廃棄漏れ）
```

## 10. 移行計画

### 10.1 既存ISO 27001情報資産台帳からの移行

| ステップ | 作業内容 | 期間 |
|---------|---------|------|
| 1 | 既存台帳のバックアップ | 1日 |
| 2 | 新規項目の追加（DB拡張） | 1週間 |
| 3 | 記録種別の分類・割当 | 2週間 |
| 4 | 保持期間の設定（記録保持スケジュール参照） | 2週間 |
| 5 | 自動計算ロジックの実装 | 1週間 |
| 6 | テスト・検証 | 1週間 |
| 7 | 本番移行 | 1日 |

### 10.2 移行時の注意点

- 既存の資産IDは変更しない（記録IDとして継続使用）
- セキュリティ分類と記録種別のマッピングを慎重に行う
- 過去の変更履歴は保持する
