# 文書管理台帳 項目定義表

## 1. 概要

文書管理台帳は、組織で作成・受領した個々の文書・記録を登録・管理するための台帳です。1件1件の文書を記録として捕捉し、その所在や状態を追跡します。

## 3. 必須レベルの説明

- **◎ 必須**：すべての文書で必ず入力が必要
- **△ 条件付き必須**：特定の条件下で必須（媒体による）
- **○ 任意**：必要に応じて入力

## 4. 入力ルール

### 4.1 文書IDの採番ルール例
- フォーマット：`DOC-[年度]-[連番4桁]`
- 例：DOC-2025-0001、DOC-2025-0002...

### 4.2 保存期間起算日の考え方
- 通常の文書：作成日
- 契約書：契約終了日
- 会計帳簿：会計年度末日
- プロジェクト文書：プロジェクト完了日

### 4.3 媒体別の入力要件

**紙のみの場合**
- 保管場所（物理）：必須
- 保管場所（電子）：不要
- ファイルパス：不要

**電子のみの場合**
- 保管場所（物理）：不要
- 保管場所（電子）：必須
- ファイルパス：必須

**紙+電子の場合**
- 保管場所（物理）：必須
- 保管場所（電子）：必須
- ファイルパス：必須

## 5. 運用上の注意点

1. **一貫性の確保**：文書種別は必ず保存年限表に定義されたものから選択すること
2. **定期的な更新**：状態が変わった場合（承認完了、廃棄等）は速やかに更新すること
3. **廃棄予定日の管理**：システムで自動計算し、期限が近づいたらアラートを出す仕組みを推奨
4. **アクセス制限の遵守**：機密文書は適切なアクセス制限を設定すること
5. **バージョン管理**：改訂があった場合は関連文書IDで旧版とリンクすること

## 6. システム化における推奨事項

- 文書IDは自動採番
- 登録日・登録者は自動取得
- 廃棄予定日は保存期間起算日と保存期間から自動計算
- 文書種別選択時に保存期間を自動入力
- 媒体選択に応じて入力必須項目を動的に変更
- 廃棄予定日が近づいたら担当者にアラート通知
