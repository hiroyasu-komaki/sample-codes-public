# 記録管理（MSR）基本編

## 記録管理の国際規格（MSR）の理解

*MSR（Management System for Records）= ISO 30301

記録管理は3つのISO規格を使って段階的に構築します：
- **ISO 30301（MSR）**: ポリシー策定（マネジメントシステムの基盤）
- **ISO 15489**: 業務プロセス設計（記録管理の実務）
- **ISO 16175**: システム化（デジタル記録の管理）

```Mermaid

        <<Management System>>
        +経営層コミットメント
        +記録管理ポリシー

        +continueImprovement()
        +establishPolicy()
        +assignResponsibility()
    }
    
    class ISO_15489 {
        <<Principles & Practice>>

    note for ISO_16175 "第3段階：システム実装<br>技術レベルのデジタル化"
```

#### 第1段階
## ポリシー層：ISO 30301（MSR）で基盤構築

ISO 30301（MSR）は「記録のためのマネジメントシステム」の要求事項を定めた、**ISO 9001型のマネジメントシステム規格⭐️** です。[7][1]

この段階では、経営層の責任とガバナンスを明確にし、記録管理を組織戦略とコンプライアンスの枠組みの中に位置付けます。[8][1]

⭐️「ISO 9001型のマネジメントシステム規格」という点が記録管理を導入する上で超重要なので、後で詳しく説明します。

#### 実施内容

**経営層のコミットメント確保**
- 組織の文脈（事業目的・利害関係者・法令要求）を踏まえ、トップマネジメントがMSRの方針と目的を承認し、資源を割り当てる。[9][1]
- 内部監査やマネジメントレビューを通じて、記録管理の有効性を継続的に評価・改善する仕組みを要求する。[1][7]

**記録管理責任者の任命**
- MSRの計画・運用・評価を統括する責任者（Record Manager等）を定め、権限と責任を文書化することが示唆される。[2][8]
- 方針の実施状況を経営層に報告し、改善を提案できる立場として位置付ける。[2]

**記録管理ポリシーの策定**
- 記録の目的、適用範囲、原則（真正性・完全性・利用性など）を示した記録管理ポリシーを策定し、全社に周知する。[6][8][2]
- 法規制・リスク・業務ニーズを踏まえ、保持期間やアクセス制御など上位方針レベルの要求を定める。[10][11]

#### 作成される成果物

この段階の成果物は、マネジメントシステムの基盤を証明する上位文書群です。経営層のコミットメントを可視化し、全社的な枠組みを示します。

- **経営層コミットメント文書**: トップメッセージや承認署名付きのMSR方針声明、リソース配分計画
- **記録管理責任者任命書**: 職務記述書（JD）、組織図への記載、権限委任マトリクス
- **記録管理ポリシー**: 適用範囲、原則（真正性・利用性等）、目的・目標、レビュー予定を明記した文書

#### 第2段階
## ビジネス要件層：ISO 15489でプロセス詳細化

ISO 15489-1:2016は、どのように記録を生成・捕捉・管理するかという「概念・原則と実務プロセス」の標準で、全ライフサイクルを対象にしています。[5][12][6]

この段階では、30301で決めた方針を、業務単位の具体的プロセス・手順に落とし込みます。[12][2]

#### 実施内容

**業務文脈分析と記録要件の特定**
- 業務プロセス、リスク、法令等を分析し、「どの業務で、どんな記録を、どの程度の期間、どの水準で保持すべきか」を要件として定義する。[6][12]
- 真正性・信頼性・完全性・利用性を満たすためのメタデータや分類・セキュリティ要件を決める。[12][6]

**各部門担当者の配置と教育**
- 各部門に記録管理の担当者やレコードコーディネーターを置き、標準化された手順に沿って記録を作成・登録する役割を担わせることが推奨される。[2][6]
- 役割と責任、手順（登録、分類、保存、廃棄）に関する教育・トレーニングが必要とされる。[6][12]

**記録ライフサイクル管理の実装**
- 記録の作成／取得、分類、維持・利用、保存、最終処分（廃棄・移管）までの一連のプロセスを設計し、保持スケジュールや処分手順を定める。[13][14][6]
- 各プロセスでのコントロール（完全性確保、アクセス管理、処分承認など）を明確にし、手順書・ワークフローとして定着させる。[13][12]

#### 作成される成果物

業務プロセスを具体化した運用レベルの文書群で、要件定義と手順を明確化します。ライフサイクル全体のコントロールを文書化し、各部門で共有可能です。

- **業務文脈分析・記録要件文書**: 業務プロセスマップ、リスク評価表、記録保持スケジュール（保存期間・処分基準）、メタデータ仕様
- **部門担当者配置・教育資料**: 役割責任マトリクス（RACI）、トレーニングカリキュラム・記録、資格証明書
- **記録ライフサイクル手順書**: 作成・捕捉・分類・保存・利用・処分フローチャート、ワークフロー、コントロールチェックリスト

#### 第3段階：
## システム要件層：ISO 16175でシステム実装

ISO 16175（特に16175-1:2020、および旧16175-2）は、デジタル記録を管理するソフトウェアに求められる機能要件とガイダンスを提供する規格です。[3][4][15]

この段階では、15489で定義したプロセスを支えるITシステムの要件定義・選定・構築を行います。[4][3]

#### 実施内容

**IT部門によるシステム構築**
- デジタル記録管理システムや業務アプリケーションに対して、「記録として管理すべきデータ」を識別し、16175のモデル要件をもとに機能要件を定義する。[16][3][4]
- 新規構築だけでなく、既存システムの評価・改善やパッケージ選定の評価基準として活用できる。[15][3]

**デジタル記録管理の自動化**
- 生成・受領時の自動捕捉、分類・メタデータ付与、保持期間の自動計算・満期通知など、記録管理機能をアプリケーションに組み込むことが意図されている。[3][4]
- 業務利用と記録管理を一体化させることで、現場負荷を抑えつつ15489・30301の要求に沿った運用を実現できる。[4][15]

**監査証跡機能の実装**
- 記録へのアクセス、変更、移管、廃棄などのイベントを追跡・ログ化する監査証跡機能が、16175の機能要件に含まれる。[15][3]
- この監査証跡は、MSRの内部監査や外部監査、法的証拠性の確保のための重要な技術的コントロールとなる。[1][3][15]

#### 作成される成果物

システム実装の技術仕様と検証証拠で、自動化・監査対応を保証します。機能要件を満たすITアーティファクトとして活用されます。

- **ITシステム構築仕様書**: 機能要件定義書（16175モデル準拠）、システムアーキテクチャ図、テスト計画・結果報告
- **デジタル記録自動化設定**: 自動捕捉ルール、メタデータ自動付与スクリプト、保持期間計算ロジック、運用マニュアル
- **監査証跡機能実装証拠**: ログ仕様書（アクセス・変更追跡）、サンプルログ出力、真正性検証レポート、認証証明（任意）

### ISO 30301（MSR）の位置づけ

ISO 30301（MSR）はISO 9001型として、経営層コミットメント（条項5）とポリシー策定（条項5.2）を第1段階の基盤とし、内部監査（条項9.2）で成果物を検証します。

この構造により、記録管理を事業戦略と統合し、認証取得が可能です。

### ISO 27001との共通構造

情報セキュリティ（ISMS）のISO 27001もISO 9001型の共通構造（附属書SL）を採用しており、記録管理のISO 30301（MSR）と同様のPDCAベースのマネジメントシステム規格です。

記録管理（MSR）とセキュリティ（ISO 27001）は基盤構造が同一のため、統合運用が容易です。

## 「統合運用が容易」とは何か

ISO 9001をベースにした規格（MSR、ISO 27001など）は、すべて**「条項6：計画」**という共通の章を持っています。ここで「何をリスクと考えるか」「どんな目標を立てるか」を決めますが、規格ごとに**焦点が違います**。

### 共通の計画プロセス（すべてのISO 9001型規格）

どの規格も以下の流れは同じです：

#### ステップ1：状況を理解する（条項4）
- 自分の組織はどんな環境にいるか？
- 誰が利害関係者か？（顧客、規制当局など）

#### ステップ2：リスクと機会を特定する（条項6.1）
- 何がうまくいかない可能性があるか？（リスク）
- どんなチャンスがあるか？（機会）

#### ステップ3：目標と計画を立てる（条項6.2）
- 具体的な目標を設定
- 誰が、いつまでに、何をするか決める

### 記録管理（MSR）の場合

**何をリスクと考えるか？**
- 重要な記録が消えてしまう
- 記録が改ざんされる
- 法律で保存が義務付けられている記録を捨ててしまう

**計画で決めること：**
- この記録は何年保存するか（保持期間）
- いつ・どうやって廃棄するか
- 記録の品質をどう保証するか（真正性、信頼性）

**例：**
契約書は7年保存、人事記録は退職後5年保存、など

### 情報セキュリティ（ISO 27001）の場合

**何をリスクと考えるか？**
- 情報が漏れる
- 不正アクセスされる
- システムが攻撃される

**計画で決めること：**
- どの情報資産にどんな脅威があるか評価
- Annex Aの93個のコントロール（対策リスト）から必要なものを選ぶ
- 適用宣言書（SoA）を作成

**例：**
パスワードポリシー、暗号化、アクセス制御など

### 2つの規格比較表

| 項目 | MSR | ISO 27001 |
|------|-----------|-----------|
| **心配すること** | 記録が消えたり改ざんされること | 情報が盗まれたりシステムが壊されること |
| **作る文書** | 保持スケジュール表 | リスク評価書、対策リスト |
| **特徴的な作業** | 各記録の保存期間を決める | 93個の対策から選ぶ |

### 統合運用が可能とは何か？

両方とも**同じ構造（条項4→6→9のPDCAサイクル）**を使っているので：

- **条項4**で組織の状況を1回分析すれば、記録管理とセキュリティ両方に使える
- **条項9.2**の内部監査も一緒にできる
- 記録管理はセキュリティの一部として扱える（記録も「情報資産」の一種だから）

**実務的には：**

「この顧客情報データベースは、セキュリティ的には暗号化が必要（ISO 27001）で、記録管理的には3年保存が必要（MSR）」というように、両方の視点で同時に管理できます。
